## Selected project
Project Lucian desktop companion

## Selection reason
Lucian is the only directly accessible active codebase, and making analytics optionally persistent turns prior session telemetry into durable local command-center infrastructure.

## Objective
Add an explicit local-only persistence toggle for privacy-safe Lucian analytics so metrics can survive app restarts without automatic cloud sync or raw prompt storage.

## Work performed
- Added a local analytics persistence module that writes settings and analytics snapshots to Electron userData storage.
- Wired analytics persistence into the Electron main process after app readiness.
- Added IPC handlers for reading persistence status and enabling/disabling local persistence.
- Exposed persistence controls through the preload bridge.
- Added a renderer checkbox toggle and status text.
- Refreshed persistence status after each prompt so stored event counts update in the UI.
- Added CSS for the persistence control row.
- Hardened persistence path validation so settings/events filenames cannot escape the configured storage directory.
- Added Vitest coverage for disabled-by-default behavior, explicit enable/write behavior, non-destructive disable behavior, and path safety.
